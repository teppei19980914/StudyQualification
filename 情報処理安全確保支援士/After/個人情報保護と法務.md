## 個人情報

**生存する個人に関する情報**であり、下記2つを表すものです。

1. 氏名、生年月日等により特定の個人を識別できるもの
2. 他の情報と照合することで個人を識別できるもの

### 個人情報の分類

IPAや個人情報保護委員会は、個人情報を以下の3分類で説明します。

1. 特定の個人を識別する情報
2. 複数組み合わせて識別できる情報
3. 個人識別符号

#### 特定の個人を識別する情報

直接個人を識別できる情報です。

* 氏名
* 住所
* 生年月日
* メールアドレス
* 電話番号

#### 複数組み合わせて識別できる情報

単体では特定できなくても、組み合わせれば個人が分かるものです。

* 性別＋年齢＋所属部署
* 会員ID＋購入履歴
* Cookie＋端末識別子

#### 個人識別符号

平成27年の法改正で追加されました。
個人識別符号とは、身体情報や符号など、単体で個人を識別できる符号のことを指します。
例えば、顔認証データや指紋/静脈パターン、マイナンバーや運転免許証番号、旅券番号、基礎年金番号などが該当します。

## 個人情報漏えい

個人情報が本人の意図しない形で、第三者に取得/閲覧/使用されることです。
漏えいは、見られただけが該当するのではなく、**外部流出の可能性が生じただけでも漏えい**とみなします。
つまり、USB紛失なども漏えいインシデントに該当します。

### 影響

個人情報漏えいによる影響は主にCIAのうち機密性の侵害になりますが、それだけにとどまりません。

|観点|内容|
|:----|:----|
|法的リスク|個人情報保護法に基づく指導、刑事罰、行政処分|
|経営リスク|企業信用失墜、株価下落|
|財務リスク|罰金、賠償金、訴訟費用|
|事業継続リスク|サービス停止、業務停止命令|
|社会的リスク|社会的非難、企業ブランド失墜|

## リスク対応

リスクを減らすために採用される方針/手続きを決定し、実施するプロセスです。
リスクにはそれぞれ下記2種類存在します。

* 脅威
* 好機

### 脅威

プロジェクト目標にマイナスに働く不確実性を言います。
脅威に対するリスク対応としては、下記4つがあります。
**プロジェクトマネージャだけでは対応できないものは「エスカレーション」します。**

|用語|内容|
|:----|:----|
|回避|脅威の原因となる活動を やめる／排除する ことで、リスクをゼロに近づける|
|低減|発生確率や影響を 減らすための対策を講じる|
|転嫁|脅威が発生した際の 損失（影響）を第三者に移す|
|受容|残存リスクを 許容範囲として認める|

### 好機

プロジェクト目標にプラスに働く不確実性を言います。
好機に対するリスクの対応としては、下記4つがあります。
**プロジェクトマネージャだけでは対応できないものは「エスカレーション」します。**

|用語|内容|
|:----|:----|
|活用|好機を 確実に実現する ための対応を取る|
|強化|好機の 発生確率や影響を高める|
|共有|他者とリソースを共有し、好機を 共同で最大化する|
|受容|積極的な行動はせず、好機が発生したら受け入れる（待つ）|

## 個人情報保護の周辺知識

|用語|内容|
|:----|:----|
|残存リスク（Residual Risk）|対策を講じても必ず残るリスク|
|管理策（Annex A / JIS Q 27002）|個人情報漏えい防止の 国際的ベストプラクティス|
|マルチレイヤ防御（多層防御）|技術的/組織的/物理的/人的対策が必須|
|技術的対策|アクセス制御/暗号化/ログ監査/EDRやウィルス対策/データマスキング|
|組織的対策|情報管理ポリシー/情報取り扱い規程/委託先管理/定期監査や内部統制|
|物理的対策|入退室管理/鍵管理/サーバ室監視/書類の施錠保管|
|人的対策|教育や訓練/誓約書/内部不正防止|

## 個人情報保護法（令和2年改正/令和4年施行）

### 個人情報保護法の目的

個人情報の **適正な取扱いに関するルール** を定め、個人の権利利益を保護しつつ、個人情報の有用性に配慮することを目的としています。

### 個人情報取扱事業者の義務（試験頻出）

| 義務 | 条文 | 内容 |
|---|---|---|
| **利用目的の特定** | 第17条 | できる限り具体的に利用目的を特定 |
| **利用目的の通知・公表** | 第21条 | 取得時に利用目的を本人に通知または公表 |
| **利用目的による制限** | 第18条 | 特定した利用目的の達成に必要な範囲でのみ取り扱う |
| **適正な取得** | 第20条 | 偽りその他不正な手段による取得の禁止 |
| **安全管理措置** | 第23条 | 漏えい等防止のために必要かつ適切な措置を講じる |
| **従業者の監督** | 第24条 | 従業者に個人データを取り扱わせる場合の監督 |
| **委託先の監督** | 第25条 | 個人データの取扱いを委託する場合の監督 |
| **第三者提供の制限** | 第27条 | 本人の同意なく第三者に個人データを提供してはならない |
| **開示・訂正・利用停止請求** | 第33-35条 | 本人からの請求に応じる義務 |

### 要配慮個人情報（第2条第3項）

**要配慮個人情報** とは、本人に対する **不当な差別、偏見その他の不利益が生じないよう特に配慮を要する個人情報** です。

| 該当する情報の例 |
|---|
| 人種 |
| 信条（宗教・思想） |
| 社会的身分 |
| 病歴 |
| 犯罪の経歴 |
| 犯罪被害の事実 |
| 身体障害・知的障害・精神障害等 |
| 健康診断等の結果 |

**試験ポイント**：要配慮個人情報は、通常の個人情報と異なり **オプトアウトによる第三者提供が禁止** されています。取得には **本人の同意が必須** です。

### オプトインとオプトアウト（試験頻出）

| 方式 | 説明 |
|---|---|
| **オプトイン** | **事前に本人の同意を得てから** 個人情報を利用・提供する方式 |
| **オプトアウト** | あらかじめ利用・提供することを通知し、**本人が拒否しない限り** 同意があったとみなす方式 |

#### オプトアウトの要件（第27条第2項）

- 個人情報保護委員会への **届出** が必要
- 本人に **通知または容易に知り得る状態** に置く
- **要配慮個人情報はオプトアウト不可**
- 不正取得された個人データもオプトアウト不可

### 匿名加工情報と仮名加工情報（試験頻出）

| 分類 | 定義 | 本人同意なしの第三者提供 | 本人への通知義務 |
|---|---|---|---|
| **匿名加工情報** | 特定の個人を識別できないように加工し、復元できないようにした情報 | **可能**（一定の規律の下） | なし |
| **仮名加工情報**（令和2年改正で新設） | 他の情報と照合しない限り特定の個人を識別できないように加工した情報 | **原則不可** | なし |
| 個人情報（元データ） | 特定の個人を識別できる情報 | 原則本人同意が必要 | あり |

**試験ポイント**：匿名加工情報は「復元不可能」なレベルまで加工するため第三者提供可能。仮名加工情報は「照合すれば復元可能」なため第三者提供は原則不可ですが、内部での分析利用が目的であり、利用目的の変更が可能という利点があります。

### 漏えい等の報告義務（令和4年施行：試験頻出）

令和2年改正により、個人データの漏えい等が発生した場合の **報告義務と本人通知義務が法的義務化** されました（従来は努力義務）。

#### 報告が必要な事態（個人情報保護委員会規則）

| 事態 | 例 |
|---|---|
| **要配慮個人情報の漏えい** | 病歴・犯歴等の流出 |
| **財産的被害のおそれ** | クレジットカード番号、口座番号の流出 |
| **不正アクセスによる漏えい** | サイバー攻撃による個人データの流出 |
| **1,000人超の漏えい** | 大規模な個人情報漏えい |

#### 報告先と期限

| 報告 | 期限 | 内容 |
|---|---|---|
| **速報** | 事態を知った日から **3～5日以内** | 概要・漏えい件数・原因等 |
| **確報** | 事態を知った日から **30日以内**（不正アクセスの場合60日以内） | 詳細な調査結果・再発防止策 |
| **本人通知** | 速やかに | 漏えいの事実・対応策等 |

### 越境移転（第28条）

外国にある第三者への個人データの提供には、以下のいずれかが必要です。

| 要件 | 説明 |
|---|---|
| **本人の同意** | 外国にある第三者への提供について同意を得る |
| **基準適合体制** | 個人情報保護委員会が認めた基準を満たす体制の整備 |
| **十分性認定** | 個人情報保護委員会が認定した国・地域（EU等） |

### 個人情報保護委員会

| 項目 | 内容 |
|---|---|
| 位置づけ | 内閣府の外局として設置された独立した機関 |
| 権限 | 個人情報取扱事業者への **報告徴収、立入検査、指導、勧告、命令** |
| 命令違反 | **1年以下の懲役または100万円以下の罰金** |
| 法人重課 | 法人に対しては **1億円以下の罰金** |

## 特許権

特許法(第1条/第68条)によると、発明を保護・利用を図るために、特許権者は、特許発明の実施を独占できる権利を持つと定義されています。
また、特許権者は「業として特許発明を実施する権利を専有する」とも定義されています。
**「実施」とは、**  製造・使用・譲渡・貸出・輸出入などを指します。

### 特許の対象

特許法(第2条)によると、自然法則を利用した技術的思想の創作のうち高度なものと定義されており、特許対象は「技術的アイデア」と解釈ができます。

#### 特許の対象外

下記は特許の対象外です。

* 単なる発見(例：数学公式)
* 精神的活動(例：占いの方法)
* 薬味の調合のような「ノウハウ」で技術性が不足するもの
* 自然法則を利用しないもの(例：経営方法)

### 特許を受けるための要件

特許法(第29条)によると、特許を受けるための要件は下記3点です。

1. 新規性
    → 発明が世界のどこにも公開されていないこと
2. 進歩性
    → 既存の技術から容易に発明できないこと
3. 産業上の利用可能性
    → 工業/産業で利用できること

### 特許権の効力

特許法(第68条/第70条)によると、特許権者は独占排他権(実施独占)を持ちます。
独占排他権によって次を禁止できます。

* 無断製造
* 無断販売
* 無断使用
* 無断輸入

#### 効力の範囲

効力の範囲は、属地性の原則から日本国内のみです。
**海外では別途特許を取得しないと保護されません。**

### 存続期間

特許法(第67条)によると、原則出願日から20年が特許権が存続する期間です。
例外として、医薬品などの一定分野では、最大5年の延長(25年)が可能です。

### 特許権とソフトウェア

ソフトウェアでも、以下の要件を満たす場合は特許対象となります。

* 技術的思想が含まれる
* 自然法則を利用している
* 実際のハードウェア動作を伴う

#### 特許となる例

ソフトウェアの特許として、下記の場合は対象となる例です。

* 画像圧縮アルゴリズム
* 暗号方式の改良
* 通信プロトコルの高速化
* AI推論の高速化手法

#### 特許とならない例

ソフトウェアの特許として、下記の場合は対象外となる例です。

* 単なるプログラムのコード
* UI設計
* 事務処理ルール
* ビジネスモデル(BM)単体
    **※ 技術的手段と結びつく場合のみ有効です。**

## 知的財産権の周辺知識

### 実用新案との違い

|分類|特許|実用新案|
|:----|:----|:----|
|対象|高度な発明|小規模改良|
|審査方式|審査あり|無審査（方式審査）|
|保護期間|20年|10年|
|進歩性|高 → 必須|低くても可|

### そのほか権利との比較

|権利|内容|例|
|:----|:----|:----|
|特許権|技術の独占|発明・技術手法|
|意匠権|物品の形状・デザイン保護|スマホ形状|
|商標権|ブランド・ロゴ保護|Appleのロゴ|
|著作権|作品保護|ソースコード、文書|

### 特許権の制限

特許法(第69条)によると、研究試験用途は権利侵害にあたらないです。
また、特許法(第79条)によると、出願前から使用している者は継続利用可能です。

### 侵害の種類

権利の侵害の種類は下記3点です。

* 直接侵害
* 間接侵害(補助的部品の提供)
* 均等論(特許請求の範囲と実質が同じ場合)

## 不正アクセス禁止法（不正アクセス行為の禁止等に関する法律）

### 不正アクセス禁止法の目的

**電気通信回線を通じた不正アクセス行為** を禁止し、電気通信に関する秩序の維持に寄与することを目的としています。

### 不正アクセス行為の定義（第2条第4項：試験頻出）

不正アクセス行為とは、**アクセス制御機能を有するコンピュータに対し、他人の識別符号（ID/パスワード等）を入力し、または脆弱性を攻撃して、本来の利用権限を超えてアクセスする行為** です。

| 類型 | 内容 | 例 |
|---|---|---|
| **識別符号の不正入力** | 他人のID/パスワードを無断で使用 | 他人のアカウントでログイン |
| **セキュリティホール攻撃** | 脆弱性を悪用してアクセス制御を回避 | SQLインジェクション、バッファオーバーフロー |

### 禁止される行為と罰則

| 行為 | 条文 | 罰則 |
|---|---|---|
| **不正アクセス行為** | 第3条 | **3年以下の懲役または100万円以下の罰金** |
| **他人の識別符号の不正取得** | 第4条 | 1年以下の懲役または50万円以下の罰金 |
| **不正アクセスを助長する行為**（ID/パスワードの不正提供） | 第5条 | 1年以下の懲役または50万円以下の罰金 |
| **他人の識別符号の不正保管** | 第6条 | 1年以下の懲役または50万円以下の罰金 |
| **フィッシング行為**（識別符号の入力を求める行為） | 第7条 | 1年以下の懲役または50万円以下の罰金 |

### 試験で注意すべきポイント

- 不正アクセス禁止法は **ネットワーク経由のアクセスのみ** が対象（端末を直接操作する場合は対象外）
- 自社管理のサーバであっても、**アクセス権限のないシステムへの不正ログイン** は対象
- **パスワードを推測しても、ログインしなければ** 不正アクセス行為には該当しない
- フィッシングサイトの **開設行為自体** が処罰対象（第7条）

### アクセス管理者の義務（第8条）

アクセス管理者は、**不正アクセスを防御するための措置を講じる努力義務** を負います。

- ID/パスワードの適切な管理
- アクセス制御機能の適切な設定
- 不正アクセスの検知・防止措置

## 不正競争防止法

### 不正競争防止法の目的

事業者間の公正な競争を確保するため、**営業秘密の侵害、技術的制限手段の無効化等の不正競争行為** を規制する法律です。

### 営業秘密の3要件（試験頻出）

営業秘密として法的保護を受けるには、以下の **3つの要件をすべて満たす** 必要があります。

| 要件 | 内容 | 具体的な判断基準 |
|---|---|---|
| **秘密管理性** | 秘密として管理されていること | アクセス制限、「社外秘」等の表示、施錠管理 |
| **有用性** | 事業活動に有用な情報であること | 技術情報、顧客リスト、製造ノウハウ等 |
| **非公知性** | 公然と知られていないこと | 一般に入手不可能な情報 |

**試験ポイント**：3要件のうち **秘密管理性** が最も争点になりやすいです。「社外秘」と表示しただけでは不十分な場合があり、**アクセス制限の実施（物理的・技術的管理）** が求められます。

### 営業秘密の侵害行為

| 行為類型 | 内容 |
|---|---|
| **不正取得** | 窃取、詐欺、脅迫等の不正手段による取得 |
| **不正使用・開示** | 不正取得した営業秘密の使用・第三者への開示 |
| **信義則違反** | 正当に取得した営業秘密を、図利加害目的で使用・開示 |
| **転得者の悪意取得** | 不正取得された情報と知りながらの取得・使用 |

### 営業秘密の罰則

- **10年以下の懲役または2,000万円以下の罰金**（個人）
- **法人重課：10億円以下の罰金**（法人）
- 海外流出の場合はさらに重い罰則

### 限定提供データ（2018年改正で新設）

**限定提供データ** とは、業として特定の者に提供する情報であって、電磁的方法で相当量蓄積・管理されているものです。

| 要件 | 説明 |
|---|---|
| **限定提供性** | 特定の者に対してのみ提供 |
| **電磁的管理性** | ID/パスワード等で管理 |
| **相当蓄積性** | 相当量のデータが蓄積 |

ビッグデータ等のデータの不正取得・使用を規制するために新設されました。

### 技術的制限手段の回避（第2条第18項）

- コピーガード、アクセス制御等の **技術的制限手段を回避する装置・プログラムの提供** を禁止
- DRM（デジタル著作権管理）の回避行為も対象

## サイバーセキュリティ基本法

### サイバーセキュリティ基本法の概要

2014年に制定された、**日本のサイバーセキュリティ政策の基本となる法律** です。

### 主要な規定

| 項目 | 内容 |
|---|---|
| **目的** | サイバーセキュリティに関する施策を総合的かつ効果的に推進 |
| **サイバーセキュリティの定義** | 情報の漏えい、滅失、毀損の防止等、安全管理のために必要な措置が講じられ、その状態が適切に維持管理されていること |
| **NISC** | 内閣サイバーセキュリティセンター（国の司令塔機能） |
| **サイバーセキュリティ戦略本部** | 内閣に設置。戦略の策定、重大インシデントの評価 |

### 各主体の責務

| 主体 | 責務 |
|---|---|
| **国** | サイバーセキュリティに関する総合的な施策の策定・実施 |
| **地方公共団体** | 地域の実情に応じた施策の策定・実施 |
| **重要インフラ事業者** | サイバーセキュリティの確保に努める |
| **サイバー関連事業者** | 自主的・積極的にサイバーセキュリティの確保に努める |
| **国民** | サイバーセキュリティの確保に必要な注意を払う |

## 情報処理安全確保支援士制度（情報処理促進法）

### 制度の概要

**情報処理安全確保支援士（登録セキスペ）** は、情報処理の促進に関する法律に基づく **国家資格（名称独占資格）** です。

### 支援士の義務（試験頻出）

| 義務 | 内容 | 罰則 |
|---|---|---|
| **信用失墜行為の禁止** | 支援士の信用を傷つける行為の禁止 | 登録の取消し等 |
| **秘密保持義務** | 業務上知り得た秘密の漏洩禁止 | **1年以下の懲役または50万円以下の罰金** |
| **講習の受講** | 毎年のオンライン講習＋3年ごとの実践講習 | 登録の取消し等 |

**試験ポイント**：秘密保持義務は **登録の取消し後も継続** します。また、名称独占資格であるため、登録していない者が「情報処理安全確保支援士」を名乗ることは **禁止** されています（1年以下の懲役または50万円以下の罰金）。

## 著作権法（詳細）

### 著作権法の概要

著作権法は、**著作物（思想又は感情を創作的に表現したもの）** を保護する法律です。特許権等の産業財産権と異なり、**登録なしに（創作と同時に）権利が発生** します。

### 著作権の構成

| 権利 | 内容 |
|---|---|
| **著作者人格権** | 公表権、氏名表示権、同一性保持権（**譲渡不可**） |
| **著作財産権（著作権）** | 複製権、公衆送信権、翻案権等（**譲渡可能**） |

### プログラムの著作物（試験頻出）

| 項目 | 内容 |
|---|---|
| **保護対象** | プログラム（ソースコード・オブジェクトコード） |
| **保護されないもの** | プログラム言語、規約（プロトコル）、アルゴリズム（第10条第3項） |
| **保護期間** | 著作者の死後70年（法人著作物は公表後70年） |

**試験ポイント**：プログラムのアルゴリズムやプログラミング言語自体は著作権で保護されません。アルゴリズムを保護したい場合は **特許** を取得する必要があります。

### 職務著作（法人著作：第15条）

以下の要件を **すべて** 満たす場合、著作者は法人等の使用者となります。

1. 法人等の **発意** に基づくこと
2. 法人等の **業務に従事する者** が作成すること
3. **職務上** 作成すること
4. 法人等の **名義** で公表すること（プログラムの場合は不要）
5. 契約・勤務規則等に **別段の定めがない** こと

### 著作権の制限（権利侵害とならない場合）

| 制限事由 | 条文 | 内容 |
|---|---|---|
| **私的複製** | 第30条 | 個人的・家庭内での使用目的の複製 |
| **引用** | 第32条 | 公正な慣行に合致し、報道等の目的上正当な範囲 |
| **プログラムのバックアップコピー** | 第47条の3 | 所有者による複製物の作成 |
| **試験問題としての複製** | 第36条 | 入学試験等での出題目的 |
| **リバースエンジニアリング** | 第30条の4 | プログラムの調査・解析（2018年改正で明確化） |

### 著作権と特許権の比較（試験頻出）

| 項目 | 著作権 | 特許権 |
|---|---|---|
| 権利発生 | **無方式主義（自動発生）** | 出願・審査・登録が必要 |
| 保護対象 | 表現（コード、文書等） | **技術的思想（アイデア）** |
| 保護期間 | 死後70年 | 出願から20年 |
| 登録 | 不要 | 必要 |
| 独立創作 | 偶然同じ表現でも侵害にならない | 独立に発明しても侵害になる |

## 電子署名法（電子署名及び認証業務に関する法律）

### 電子署名法の概要

電子署名に **手書き署名や押印と同等の法的効力** を認め、電子商取引等の信頼性を確保するための法律です。

### 電子署名の要件（第2条）

電子署名として認められるための要件は以下の2点です。

| 要件 | 説明 |
|---|---|
| **本人性** | 当該電磁的記録に記録された情報が **当該措置を行った者の作成に係るもの** であることを示すもの |
| **非改変性** | 当該電磁的記録に記録された情報について **改変が行われていないかどうか** を確認できるもの |

### 電子署名の法的効力（第3条：試験頻出）

**電磁的記録に本人による電子署名が行われている場合、真正に成立したものと推定** されます。

これは民事訴訟法上の「二段の推定」に相当し、紙文書における「本人の印鑑による押印」と同等の推定力を持ちます。

### 認定認証業務

| 項目 | 内容 |
|---|---|
| **認定認証業務** | 主務大臣の認定を受けた認証局が行う電子証明書の発行業務 |
| **特定認証業務** | 電子署名の本人確認を行う認証業務 |
| **認定の基準** | 本人確認の方法、設備、業務体制等に関する技術基準への適合 |

## 刑法（コンピュータ犯罪関連条文）

### コンピュータ犯罪に関する刑法規定

| 罪名 | 条文 | 内容 | 罰則 |
|---|---|---|---|
| **不正指令電磁的記録に関する罪（ウイルス作成罪）** | 第168条の2/3 | マルウェアの **作成、提供、供用、取得、保管** | 3年以下の懲役または50万円以下の罰金 |
| **電子計算機使用詐欺罪** | 第246条の2 | 虚偽のデータ入力等による不正な財産上の利益の取得 | 10年以下の懲役 |
| **電子計算機損壊等業務妨害罪** | 第234条の2 | コンピュータの破壊・不正データ入力による業務妨害 | 5年以下の懲役または100万円以下の罰金 |
| **電磁的記録不正作出罪** | 第161条の2 | 権限なく電磁的記録を作成・変更 | 5年以下の懲役または50万円以下の罰金 |
| **不正電磁的記録供用罪** | 第161条の2第3項 | 不正に作出された電磁的記録を使用 | 5年以下の懲役または50万円以下の罰金 |

2011年の刑法改正で新設された規定です。

| 行為 | 罰則 |
|---|---|
| **作成・提供** | 3年以下の懲役または50万円以下の罰金 |
| **供用（実行させる行為）** | 3年以下の懲役または50万円以下の罰金 |
| **取得・保管** | 2年以下の懲役または30万円以下の罰金 |

**試験ポイント**：「不正指令電磁的記録」とは、**正当な理由がないのに、人のコンピュータにおける実行の用に供する目的** で作成されたプログラムです。セキュリティ研究目的での保有は「正当な理由」として認められる場合があります。

## GDPR（EU一般データ保護規則：General Data Protection Regulation）

### GDPRの概要

2018年5月に施行された、**EU域内の個人データ保護に関する規則** です。日本の個人情報保護法よりも厳格な規制を持ちます。

### GDPRの主要原則

| 原則 | 内容 |
|---|---|
| **適法性、公正性、透明性** | データ処理は適法かつ公正で、本人に透明でなければならない |
| **目的の限定** | 明確かつ正当な目的のためにのみ収集 |
| **データの最小化** | 目的に必要な最小限のデータのみ収集 |
| **正確性** | 正確かつ最新の状態を保つ |
| **保存の制限** | 目的に必要な期間のみ保存 |
| **完全性と機密性** | 適切なセキュリティ対策の実施 |

### GDPRの主要規定（試験頻出）

| 規定 | 内容 |
|---|---|
| **域外適用** | EU域内の個人データを処理する場合、EU域外の企業にも適用 |
| **忘れられる権利（削除権）** | データ主体が自身のデータの削除を要求する権利 |
| **データポータビリティ権** | 自身のデータを他の管理者に移行する権利 |
| **DPO（データ保護オフィサー）** | 一定の場合にDPOの選任が義務 |
| **データ侵害通知** | 個人データ侵害の発生から **72時間以内** に監督機関に通知 |
| **DPIA（データ保護影響評価）** | 高リスクのデータ処理に先立つ影響評価の実施 |

### GDPRの罰則

| 違反レベル | 制裁金の上限 |
|---|---|
| 軽度の違反 | **1,000万ユーロまたは全世界年間売上高の2%** のいずれか高い方 |
| 重度の違反 | **2,000万ユーロまたは全世界年間売上高の4%** のいずれか高い方 |

### 日本とGDPRの関係

- 2019年1月に **十分性認定** を相互に取得（日EU間での個人データの自由な移転が可能）
- 日本企業がEU域内の個人データを扱う場合はGDPRの遵守が必要

### 日本の個人情報保護法とGDPRの比較

| 項目 | 個人情報保護法 | GDPR |
|---|---|---|
| 適用範囲 | 日本国内 | **域外適用あり** |
| 同意 | オプトアウト可（一部） | **原則オプトイン** |
| 漏えい報告 | 3～5日以内（速報） | **72時間以内** |
| 罰則 | 最大1億円 | **最大全世界売上高の4%** |
| 忘れられる権利 | 利用停止請求 | **削除権（明示的に規定）** |
| DPO | 規定なし | **一定の場合に選任義務** |

## マイナンバー法（行政手続における特定の個人を識別するための番号の利用等に関する法律）

### マイナンバー法の概要

個人番号（マイナンバー）の適正な取扱いに関する規律を定める法律です。個人情報保護法の **特別法** として位置づけられています。

### 特定個人情報の取扱い（試験頻出）

| 項目 | 個人情報 | 特定個人情報（マイナンバー含む情報） |
|---|---|---|
| 利用目的 | 通知・公表 | **税・社会保障・災害対策の3分野に限定** |
| 第三者提供 | 本人同意があれば可能 | **法令に基づく場合のみ**（本人同意でも不可） |
| 漏えい時の罰則 | 法人：1億円以下 | **個人：4年以下の懲役または200万円以下の罰金** |
| オプトアウト | 一定の場合に可能 | **不可** |

**試験ポイント**：マイナンバーは個人情報保護法よりも **厳格な規律** が適用されます。特に「本人の同意があっても目的外利用・第三者提供は不可」という点が重要です。

### 特定個人情報保護評価（PIA：Privacy Impact Assessment）

行政機関等がマイナンバーを含む個人情報ファイルを保有する場合、**事前に特定個人情報保護評価** を実施する義務があります。

## プロバイダ責任制限法（特定電気通信役務提供者の損害賠償責任の制限及び発信者情報の開示に関する法律）

### プロバイダ責任制限法の概要

インターネット上の **権利侵害情報（名誉毀損、著作権侵害等）** に関し、プロバイダ等の損害賠償責任の制限と、発信者情報開示請求の手続きを定める法律です。

### 主な規定

| 規定 | 内容 |
|---|---|
| **損害賠償責任の制限** | プロバイダが権利侵害情報の存在を知らなかった場合、損害賠償責任を負わない |
| **発信者情報開示請求** | 権利を侵害された者は、プロバイダに対して発信者の情報開示を請求可能 |
| **発信者情報開示命令**（2022年改正） | 裁判所への非訟手続による迅速な開示命令制度の新設 |
