## 1. ISO/IEC 15408(Common Criteria：CC)  

**情報システム/製品のセキュリティ機能が "どの程度信頼できるか" を評価するための国際規格です。**  

### ■ 1.1 主旨  

- 製品(OS、FW、暗号モジュール等)が備えるセキュリティ機能を **客観的に評価**  
- 認証されたシステムは **国際相互承認(CCRA)** により海外でも通用  

### ■ 1.2 CC の三構成要素  

| 要素 | 内容 |
|------|------|
| **PP(Protection Profile)** | 製品カテゴリごとの安全要求(例：ファイアウォール PP) |
| **ST(Security Target)** | 個別製品ごとに定義された安全要求 |
| **TOE(Target of Evaluation)** | 評価対象そのもの |

### ■ 1.3 EAL(Evaluation Assurance Level)  

評価保証レベル 1〜7 が定められています。  
EAL が高いほど「検証の深度」が高いです(設計検証/テスト/形式手法など)。  

試験で問われやすいポイント：  

- **EAL は “信頼性の評価レベル” であり、製品のセキュリティ性能そのものを保証するわけではない**  

## 2. ISO/IEC 27002(情報セキュリティ管理策の実践規範)  

**ISO/IEC 27001 における「付属書Aの管理策」を具体化したガイドライン規格です。**  

### ■ 2.1 目的  

組織が導入すべき情報セキュリティ管理策を体系的に提供します。  

### ■ 2.2 管理策ドメイン構成(2022版)  

- 組織的管理策(Organizational)  
- 人的管理策(People)  
- 物理的管理策(Physical)  
- 技術的管理策(Technological)  

### ■ 2.3 代表的な管理策  

- 物理的入退室管理  
- バックアップ  
- アクセス制御  
- 暗号管理  
- ログ監視  
- セキュリティインシデント管理  
- サプライチェーンセキュリティ  

#### ✔ SC 試験ポイント  

27002 は「情報セキュリティ管理策の網羅的ガイドライン」であり、**実装方法の統一ではなく “ベストプラクティスの参照” に使う規格です。**  

## 3. ISO/IEC 27017(クラウドセキュリティ管理策)  

**クラウドサービス(IaaS/PaaS/SaaS)に特化したセキュリティ管理策です。**  
27002 をベースに、クラウド固有の問題点を補完します。  

### ■ 3.1 目的  

クラウドの **利用者(Cloud Customer)** とクラウド **サービス提供者(CSP)** の双方に必要なセキュリティ管理策を提供します。  

### ■ 3.2 代表的クラウド特有管理策  

| 管理策 | 内容 |
|--------|------|
| **責任共有モデル** | 利用者と CSP のセキュリティ責任範囲を明確化 |
| **仮想化管理策** | ハイパーバイザ保護、VM分離 |
| **管理者アクセスの保護** | API 鍵管理、多要素認証 |
| **顧客環境の分離(Multi-tenancy)** | テナント間干渉防止 |
| **ログの連携/取得範囲** | CSP が提供するログの明確化 |

#### ✔ SC 試験ポイント  

- 27002 との違い → **クラウド固有リスクに対応しているかどうか**  
- 特に SaaS/IaaS で「誰が何を守るのか？」が重要  

## 4. ISO/IEC 30147(IoT セキュリティ開発ライフサイクル)

**IoT 製品やサービスのセキュリティを、開発ライフサイクル全体で確保するための国際規格。**

### ■ 4.1 対象  

- センサー  
- IoT デバイス  
- ゲートウェイ  
- クラウド IoT サービス  

### ■ 4.2 主な要求事項  

| 開発プロセス | 内容 |
|---------------|--------|
| **脅威分析(Threat Modeling)** | STRIDE に類似、攻撃経路を洗い出す |
| **セキュア設計** | 認証/更新/暗号化を前提とした設計 |
| **セキュア実装** | ソースコードの安全性、脆弱性排除 |
| **テストと評価** | ペネトレーションテスト、更新テスト |
| **セキュアアップデート** | OTA 更新の署名検証必須 |
| **廃棄フェーズ対応** | データ消去、鍵削除 |

#### ✔ 特徴  

- IoT 製品向けの「セキュア開発プロセス」規格  
- 27017 がクラウド、30147 が IoT を補完する関係  

## 5. SC 午前Ⅱ 問9 の想定問われ方(要点)

試験で問われる観点は以下の通り：  

- **ISO/IEC 15408：セキュリティ機能の評価規格(EAL)である**  
- **ISO/IEC 27002：情報セキュリティ管理策(ガイドライン)である**  
- **ISO/IEC 27017：クラウド向けの管理策である**  
- **ISO/IEC 30147：IoT のセキュア開発工程である**

それぞれの **焦点がどこにあるかを明確に識別する力** が求められます。
