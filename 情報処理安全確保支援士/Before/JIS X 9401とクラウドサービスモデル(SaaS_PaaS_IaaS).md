## 1. JIS X 9401:2016 とは  
**JIS X 9401:2016(ISO/IEC 17788:2014 と一致)**は、クラウドコンピューティングに関する **概要と用語の標準規格** です。  

### 規格の目的  

- クラウドに関する用語の定義統一  
- 各サービスモデル(SaaS/PaaS/IaaS)の役割整理  
- クラウドエコシステム(利用者/提供者/仲介者)の明確化  
- クラウド管理/ガバナンスの基本概念提供  

## 2. クラウドサービスモデル(JIS X 9401/ISO 17788 準拠)

### 2.1 SaaS(Software as a Service)

利用者は **アプリケーションをそのまま利用** するサービスモデルです。  

#### 特徴  

- インフラ/プラットフォーム/アプリケーションすべてをプロバイダが管理  
- 利用者は設定やデータ入力のみに集中  
- 最も利用者の管理負担が少ない  

#### 例  

- Gmail、Microsoft 365、Salesforce  
- 会計ソフト、CRM、グループウェア  

#### 利用者の責任  

- アカウント管理  
- データ保護  
- アクセス制御設定  

### 2.2 PaaS(Platform as a Service)  

アプリケーションを構築する **実行基盤(OS/ミドルウェア)を提供** するサービスです。  

#### 特徴  

- 開発者は **環境構築不要でアプリ開発に集中**  
- OS、ランタイム、DB 等は提供者が管理  
- スケールアウト機能なども提供される  

#### 例  

- Google App Engine  
- Microsoft Azure App Service  
- AWS Elastic Beanstalk  

#### 利用者の責任  

- アプリケーションコード  
- データ構造/設定  
- API 利用  

### 2.3 IaaS(Infrastructure as a Service)  

**仮想サーバ、ネットワーク、ストレージなど基盤そのもの**を提供します。  

#### 特徴  

- OS/ミドルウェアは利用者が管理  
- 柔軟性が高く、オンプレと最も近い  
- セキュリティ設定の責任が大きい  

#### 例  

- Amazon EC2  
- Microsoft Azure VM  
- Google Compute Engine  

#### 利用者の責任  

- OS/パッチ管理  
- ミドルウェア設定  
- ファイアウォール設定  
- 監視/ログ管理  

## 3. 3サービスモデルの比較(責任共有モデル)

**IaaS → PaaS → SaaS の順で利用者負担が軽減** します。  
IPA試験では特に **責任範囲の違い**が問われる。

| 項目 | SaaS | PaaS | IaaS |
|------|------|------|------|
| アプリ管理 | × | △(コードのみ) | ○ |
| ミドルウェア | × | ×(提供側) | ○ |
| OS管理 | × | × | ○ |
| 仮想化層 | × | × | ×(提供側) |
| ネットワーク | × | × | × |
| データ管理 | 利用者 | 利用者 | 利用者 |

## 4. クラウドの特性(JIS X 9401/ISO 17788)

クラウドは次の5特性を満たします(NIST SP 800-145 にも一致)。  

1. **オンデマンド自動サービス(On-demand self-service)**  
2. **広範囲ネットワークアクセス(Broad network access)**  
3. **リソース共有(Resource pooling)**  
4. **迅速な弾力性(Rapid Elasticity)**  
5. **計測サービス(Measured service)**  

これらにより、従来のオンプレミスとは異なる高度なスケール/可用性を実現します。  

## 5. クラウドの導入におけるリスク(支援士頻出)

#### ■ データ管理リスク  

- データの所在(海外リージョン)  
- 法規制(GDPR、個人情報保護法)  

#### ■ 可用性リスク  
- 障害時のクラウド依存  
- 冗長構成の不足  

#### ■ セキュリティリスク  

- 設定ミス(S3 バケット公開)  
- IAM権限不備  
- API 鍵漏えい  

#### ■ 外部委託管理リスク  

- SLAの不備  
- サードパーティ(再委託)の確認不足  

## 6. 周辺知識(試験でセット問われる内容)

### ■ クラウドサービス提供者/利用者の役割(cloud service provider/cloud service customer)

JIS X 9401 ではエコシステムとして明確に定義しています。  

### ■ マルチテナンシ(Multi-tenancy)

複数利用者が同一物理基盤を共有する仕組みです。  

### ■ API セキュリティ  

IaaS/PaaS では API 認証が特に重要です。  

### ■ クラウド監査(SOC1/SOC2)  

委託先管理において必須です。  

### ■ ISO/IEC 27017(クラウドセキュリティ管理策)  

責任共有モデルを明確化しています。
