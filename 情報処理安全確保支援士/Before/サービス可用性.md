## サービス可用性(Availability)

必要な時に情報及び関連資産にアクセスできる状態を確実にすることです。  
つまり、「サービスが止まらない」「利用者が必要な時に利用できる」状態を守ることが大切です。  

### 情報セキュリティの三大要素CIAのA

|要素|意味|
|:----|:----|
|Confidentiality（機密性）|情報を秘密に保つ|
|Integrity（完全性）|改ざんされていない状態を保つ|
|Availability（可用性）|必要な時に利用できる|

### 可用性を損なう攻撃

1. Dos/DDos攻撃  
    * 大量リクエストでサーバを高付加にし、サービスを停止させる  
2. リソース枯渇(メモリ/ディスクの消費)  
    * ログの肥大化や無限ループ  
3. ハード障害  
    * HDD障害やNIC断に加え、電源障害など  
4. ランサムウェア  
    * システム暗号化により利用不可  

### 可用性を確保する手段

1. 冗長化(Redundancy)  
    1. サーバの二重化  
    2. RAID  
    3. ネットワークの多重化  
    4. DNS冗長  
2. 可用性設計(Availability Architecture)  
    1. フォールトトレランス  
    2. フェイルオーバー  
    3. ロードバランシング  
    4. クラスタリング  
3. バックアップと迅速なリストア  
    1. RTO(復旧時間目標)  
    2. RPO(復旧時点目標)  
    3. バックアップ多重保管  
4. 管理とインシデント対応  
    1. 障害監視  
    2. アラート  
    3. SLA違反検知  
5. DDos対策  
    1. CDN/WAF  
    2. 通信のレート制限  
    3. トラフィックフィルタリング  
    4. ISPレベルでの対策  

## 可用性に関連する周辺知識

1. SLA(Service Level Agreement)  
    * 稼働率(例：99.99%)  
    * 平均故障間隔(MTBF)  
    * 平均修復時間(MTTR)  
2. SPOF(Single Point of Failure)  
    * システムの単一故障点  
    * 可用性摂家における最大の敵  
3. BCP(事業継続計画)  
    * 大規模障害でも業務を継続する計画  
    * DR(Disaster Recovery)との関連  
4. クラウドの可用性  
    * リージョン/ゾーンの冗長化  
    * Multi-AZ配置  
    * CDNによる配信負荷分散  
5. データベース可用性  
    * レプリケーション  
    * フェイルオーバー  
    * シャーディング
