## 試験概要

### 2026年度からの変更点（CBT移行）

令和8年度（2026年度）より、情報処理安全確保支援士試験は**ペーパー方式からCBT（Computer Based Testing）方式に移行**します。

| 変更項目 | 変更前 | 変更後 |
|---|---|---|
| 受験方式 | 紙・鉛筆 | PC・キーボード入力 |
| 科目名称 | 午前I / 午前II / 午後 | 科目A-1 / 科目A-2 / 科目B |
| 日程 | 春・秋 各1日 | 一定期間内で日時・会場を自由選択 |
| 分割受験 | 不可 | **科目Aと科目Bを別日で受験可能** |
| 出題内容・問題数・時間 | ― | **変更なし** |

> 公式情報: [IPA CBT移行ページ](https://www.ipa.go.jp/shiken/2026/ap_koudo_sc-cbt.html)

### 試験構成・合格基準

| 科目 | 旧称 | 出題数 | 試験時間 | 形式 | 合格基準 | 備考 |
|---|---|---|---|---|---|---|
| 科目A-1 | 午前I | 30問 | 50分 | 四択 | 60点以上 | **応用情報合格者は免除可**（要申請） |
| 科目A-2 | 午前II | 25問 | 40分 | 四択 | 60点以上 | SC特有の専門知識 |
| 科目B | 午後 | 大問複数 | 150分 | 記述・選択 | 60点以上 | キーボード入力に変更 |

> **⚠️ 科目A-1免除を必ず申請すること**
> 応用情報技術者試験の合格後2年以内であれば免除対象。
> 申請先: [IPA 科目A-1免除申請ページ](https://www.ipa.go.jp/shiken/henjo/gozen1.html)

### 出題分野（科目A-2 / 科目B 共通）

| 大分類 | 主なテーマ |
|---|---|
| 情報セキュリティ基礎 | CIA・リスクマネジメント・ISMS・セキュリティポリシー |
| 暗号・認証技術 | 共通鍵/公開鍵暗号・PKI・デジタル署名・多要素認証・SSO |
| ネットワークセキュリティ | ファイアウォール・IDS/IPS・VPN・TLS・DNS/Webセキュリティ |
| サイバー攻撃・マルウェア | フィッシング・ランサムウェア・SQLインジェクション・XSS |
| セキュリティ管理・法令 | 不正アクセス禁止法・個人情報保護法・NISC・GDPR |
| システム開発・セキュア実装 | セキュアプログラミング・脆弱性診断・SDLC |
| インシデント対応・監査 | CSIRT・ログ分析・フォレンジクス・セキュリティ監査 |

## 過去問道場 活用戦略（科目A-2）

### 基本ルール

1. **毎日10〜20問** を習慣化（低活動期は10問・高活動期は20問）
2. 間違えた問題は必ず**解説を読む**（正解した問題の解説は読まなくてよい）
3. 同じ問題に**3回正解**したら卒業。3回以内は繰り返す
4. 月1回、**直近〇年分の通し演習**で正答率を記録し、推移を管理する

### 月別演習目標（累計）

| 時期 | 月間問題数 | 通し演習 | 目標正答率 |
|---|---|---|---|
| 3月 | 130問 | なし | 40%以上（感触把握） |
| 4月 | 160問 | 直近1年分 | 50%以上 |
| 5月 | 160問 | 直近2年分 | 60%以上 |
| 6月 | 245問 | 直近1年分 | 65%以上 |
| 7月 | 220問 | 直近2年分 | 70%以上 |

### 分野別 優先度

| 優先度 | 分野 | 理由 |
|---|---|---|
| ★★★ 高 | 暗号・認証・PKI | 毎年必出・配点高・科目Bにも直結 |
| ★★★ 高 | ネットワーク（TLS・VPN・FW） | Azure業務経験を活かせる |
| ★★★ 高 | サイバー攻撃手法 | 科目Bの設問パターンに頻出 |
| ★★☆ 中 | 法令・制度・ガイドライン | 暗記中心・得点源になりやすい |
| ★★☆ 中 | マネジメント・ISMS・リスク | 概念理解で安定して得点できる |
| ★☆☆ 低 | セキュアプログラミング | 配点少・深入りしない |

## 科目B 解答フレームワーク

科目B（旧 午後）はパターンを掴むことで大幅に得点が安定します。
以下のフレームを文章テンプレートとして身につけること。

### パターン①：リスク・対策型（最頻出）

```
【問】〜の場合に発生しうるリスクと、その対策を述べよ。

【答】〇〇という脅威により、△△という被害が発生するリスクがある。
     対策として、××を実施することで当該リスクを低減できる。
```

> **例**：フィッシングメールにより、利用者の認証情報が窃取されるリスクがある。
> 対策として、多要素認証（MFA）を導入することで、認証情報のみの窃取では不正アクセスを防止できる。

---

### パターン②：設計・選定型

```
【問】〇〇の目的のために適切な方式・技術を選定し、理由を述べよ。

【答】〇〇の目的を達成するため、××方式を選定する。
     理由は、△△という特性を持つためであり、□□の要件を満たせるからである。
```

---

### パターン③：調査・確認型（インシデント対応）

```
【問】インシデント発生時に確認すべき事項を述べよ。

【答】〇〇の事象を確認するため、△△のログ（またはシステム）を調査する。
     具体的には、××を確認し、□□の有無を特定する。
```

---

### パターン④：設定ミス・脆弱性指摘型

```
【問】以下の設定において問題点を指摘し、修正方法を述べよ。

【答】〇〇の設定は、△△という攻撃に対して脆弱である。
     なぜなら、□□という理由で攻撃者が××を実行できるからである。
     修正方法として、◎◎に変更することで当該脆弱性を解消できる。
```

---

### 解答時の注意点（CBT＝キーボード入力）

- **主語を明確に**：「システムが」「攻撃者が」「利用者が」
- **動詞を具体的に**：「実施する」「確認する」「設定する」「制限する」
- **理由・根拠を必ず添える**：「なぜなら〜だから」
- **キーボード入力の活用**：
  - 「脆弱性」「可用性」「認証」などは変換候補を活用（手書きより有利）
  - 文章の挿入・削除が容易なため、下書き→編集→仕上げの流れを意識
  - 文字数制限がある場合は、入力中に残り文字数を確認する習慣をつける
  - タイピング速度の目安：**100文字/分以上**（不安な場合は事前に練習）

## 補足

- **業務経験の活用**：Azureセキュリティの実務経験はネットワーク・ゼロトラスト・認証系の問題で直接活かせる。業務で経験したインシデント対応や設計判断を科目Bの解答素材にすること。
- **合格できなかった場合**：CBT方式では再受験タイミングが柔軟になるため、弱点分野を絞り込んで次のシーズンに再挑戦できる。AZ-500への移行を焦らず、SCを仕上げることを優先する。
- **SC-200との相乗効果**：SC（IPA）とSC-200（Microsoft）は知識の重複が多い（Azure Sentinelとログ分析・インシデント対応等）。SC取得後にSC-200の学習コストは大幅に下がる。
